# TL;DR

切换 IDEA 的运行时 JRE，使用最新版本即可解决问题。

方法：

在 IDEA 打开的情况下，双击 shift 打开命令控制台 -> 输入 "Choose Boot Java Boot Runtime for the IDE" -> 选择最近的 JRE 版本（我是 21）-> OK 并重启 IDEA

![](/static/blog/2024-07-02-idea-lagging/2024-07-02-17-33-40.png)

# 细节

我花了 1k 多 RMB 买了 IDEA Ultimate 的正版授权，但是用了一段时间之后，发现输入中文巨卡无比。

说一下我的情况

- 系统：macOS
- 硬件：MacBook Pro 2020 16G i5
- IDEA 版本：IDEA Ultimate 2024.1
- 输入法：鼠须管（但是用 mac 自带的输入法更卡）
- 使用两个外接屏幕

于是我在很长一段时间内，在写代码时都只用英文。但是工作原因，我需要一些中文注释。

有一天我实在不想忍了，于是尝试解决一下这个问题。

网上也有很多解决方案，比如关闭一些插件，关闭代码检查等等，我尝试了之后效果均不明显。

我曾一度怀疑是我硬件性能不够，想换电脑了。但是奈何囊中羞涩，于是还是想看看能不能在软件上解决问题。

最后，我看到一篇论坛：[发现一个可能的 IDEA 输入法卡顿的原因 - V2EX](https://hk.v2ex.com/t/447961)

其中提到：

> jetbrains已经确定了是他们的锅了，目前该问题在今天发布的2018.1.2的IDEA中还未解决。
> 具体看这里：https://confluence.jetbrains.com/display/JRE/1.8.0_152-release-1136-b29+x86_64

于是我就想到可以通过切换 JRE 来解决。就有了上述的解决方案。
